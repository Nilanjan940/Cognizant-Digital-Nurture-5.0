# Lab 1 - Understanding ORM with a Retail Inventory System

## Objective

Understand Object Relational Mapping (ORM) and the role of Entity Framework Core in modern .NET applications.

---

## What is ORM?

ORM (Object Relational Mapping) is a technique that maps C# classes to relational database tables.

### Example

| C# Class | SQL Table |
|-----------|------------|
| Product | Products |
| Category | Categories |

| Object Property | Table Column |
|-----------------|--------------|
| Product.Name | Name |
| Product.Price | Price |

---

## Benefits of ORM

### Productivity
Developers work with objects instead of writing SQL repeatedly.

### Maintainability
Business logic remains separate from database logic.

### Type Safety
Compile-time checking reduces runtime errors.

### Database Abstraction
Application code remains mostly independent of the underlying database.

## Entity Framework Core vs Entity Framework 6

| Feature | EF Core 8 | EF6 |
|----------|-----------|-----|
| Cross Platform | Yes | No |
| Linux Support | Yes | No |
| Performance | High | Moderate |
| LINQ Support | Yes | Yes |
| Modern Features | Yes | Limited |
| Cloud Ready | Yes | Partial |

---

## EF Core 8 Features

### JSON Column Mapping
Store JSON data directly in database columns.

### Compiled Models
Improves startup performance.

### Better Bulk Operations
Optimized handling of large datasets.

### Interceptors
Customize query and save behaviors.
